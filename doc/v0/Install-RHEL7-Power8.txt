QIS install process on RHEL7.1 on IBM Power8 / ppc64le

Follow the normal install documentation for:

* Memcached
* Postgres server
* QIS user account
* QIS dependencies and package files
  * Run `sudo easy_install pip` instead of `sudo yum install python-pip`
* QIS application files
* Apache server
  * Install the conf files from the `deploy/centos7` folder
  * If the Apache worker limit needs to be higher than 256, add new limits (here 500) to
    `/etc/httpd/conf/httpd.conf`:
  * ```# For prefork MPM only
       <IfModule prefork.c>
           MaxRequestWorkers 500
           ServerLimit 500
           MaxConnectionsPerChild 10000
       </IfModule>
    ```
* System limits
* selinux
  * Use the policy files from the `deploy/centos7` folder

Additional steps to create a build server:

Install development packages:

sudo yum -y install git curl wget gcc make \
           python-devel python-virtualenv openldap-devel postgresql-devel \
           ImageMagick-devel libmemcached-devel zlib-devel setroubleshoot-server

Benchmarking:

* IBM Power8E
* 80 "CPUs" from /proc/cpuinfo @ 3.4 GHz
* RHEL 7.1, stock packages:
  * Python 2.7.5
  * ImageMagick 6.7.8-9 Q16
  * Memcached 1.4.15
  * PostgreSQL 9.2.13
* QIS v1.41
